# AI 多层级记忆系统

## 概述

本记忆系统基于 Anthropic 的 Personal AI Infrastructure 设计理念，实现三层记忆架构，让 AI 能够跨会话记忆和学习。

## 核心理念

> 问题不在于 AI 的记忆力，而在于我们一直把 AI 当成了"工具"，而不是"基础设施"。

传统 AI：用完就忘，像租用计算能力
记忆系统：持久化学习，像拥有个人电脑

## 系统架构

```
.memory/
├── README.md                    # 本文件
├── MEMORY_P0.md                 # P0 热记忆（工作记忆）
├── MEMORY_P1.md                 # P1 温记忆（经验教训）
├── MEMORY_P2/                   # P2 冷记忆（历史归档）
│   ├── archive_2026-02.md       # 按月归档
│   └── ...
├── telos/                       # TELOS 系统（身份定义）
│   ├── MISSION.md               # 使命
│   ├── GOALS.md                 # 目标
│   ├── PROJECTS.md              # 项目
│   ├── LEARNED.md               # 学到的
│   ├── CHALLENGES.md            # 挑战
│   └── IDEAS.md                 # 想法
└── maintenance/                 # 维护脚本
    ├── health_check.py          # 健康检查
    └── archive_manager.py       # 归档管理
```

## 三层记忆分级

### P0 - 热记忆 (Hot Memory)
**用途**：永远在脑子里的内容
- 核心规则
- 当前任务
- 最近 7 天的关键决策
- 项目上下文

**特点**：每次会话都会加载

### P1 - 温记忆 (Warm Memory)
**用途**：需要时能想起来
- 已完成的任务
- 踩过的坑
- 学到的教训
- 最佳实践

**特点**：需要时会被检索

### P2 - 冷记忆 (Cold Memory)
**用途**：长期归档
- 30 天前的历史
- 结构化存储
- 详细日志

**特点**：长期存储，需要时查询

## 使用方法

### 会话开始时
AI 会自动读取 P0 热记忆和 TELOS 系统，了解当前上下文。

### 记录新信息
- **重要决策** → 写入 P0
- **经验教训** → 写入 P1
- **日常日志** → 写入 P2

### 自动维护
- P0 超过 200 行 → 自动归档到 P1
- P1 内容超过 90 天 → 自动归档到 P2
- P2 按月整理存储

## 记忆写入原则

1. **记教训，不只记事件** - "为什么失败"比"失败了"更重要
2. **记模式，不只记结果** - 发现规律比记录结果更有价值
3. **记原因，不只记决定** - 决策依据比决定本身更重要
4. **记变化，不只记状态** - 为什么改变比改变了什么更关键

## 与 .agent 系统的关系

- `.agent/` - 增量开发流程控制
- `.memory/` - 持久化记忆和学习

两者协同工作：
1. `.agent/` 控制开发流程
2. `.memory/` 存储学到的经验
3. 新会话时两者都会被加载
