# 学到的 (Learned)

## 核心经验

### 关于 PPT 开发

1. **代码验证 ≠ 实际效果**
   - `python-pptx` 读取的颜色值可能与 PowerPoint 实际渲染不同
   - 必须用 PowerPoint COM API 或实际打开文件验证
   - 创建了 `ppt_color_verification` skill 记录验证方法

2. **文本格式处理的复杂性**
   - Run 级别的格式需要精确处理
   - 组合形状内的文本需要递归查找
   - 颜色标记需要处理好文本分割

3. **布局计算需要考虑实际渲染**
   - 文本框高度在不同情况下计算方式不同
   - 需要考虑字体、行距、边距的综合影响

### 关于增量开发

1. **每次只做一个功能**
   - 避免一次性做太多导致混乱
   - 便于测试和验证
   - 问题更容易定位

2. **三层验证体系**
   - 代码层测试：单元测试
   - 目标环境验证：用实际软件 API
   - 用户视角验证：手动打开检查

3. **进度记录很重要**
   - agent-progress.txt 是跨会话的关键
   - 详细记录做了什么和为什么
   - 帮助快速恢复上下文

### 关于记忆系统

1. **记教训比记事件更重要**
   - "为什么失败" 比 "失败了" 更有价值
   - 记录决策依据，不只是决策结果
   - 模式识别比单点记录更有用

2. **分层管理记忆**
   - 热记忆：日常工作上下文
   - 温记忆：可复用的经验
   - 冷记忆：历史记录归档
