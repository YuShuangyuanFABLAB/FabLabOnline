上下文工程

来源： https://www.anthropic.com/engineering/context-engineering
发布日期： 2025-09-29

一、什么是上下文工程

1.1 定义

上下文工程是优化LLM输入的艺术和科学，通过精心设计上下文来提升模型性能。
核心观点：模型的能力很大程度上取决于它接收到的上下文质量。

1.2 上下文组成部分

完整上下文结构：
├── 系统提示（System Prompt）
├── 示例（Few-shot Examples）
├── 检索内容（Retrieval Context）
└── 用户输入（User Input）

二、上下文优化策略

2.1 位置优化

重要信息应放在上下文的开头或结尾，因为模型对这两个位置最敏感。

2.2 信息密度

平衡信息量和可读性，避免信息过载。

2.3 结构化格式

使用清晰的格式帮助模型理解内容结构。

案例 3

Few-shot示例优化

场景

优化Few-shot示例以提高模型性能

选择策略

案例 4

上下文缓存策略

场景

优化重复上下文的处理效率

关键设计

class ContextCache:
    def get(self, context: str) -> Optional[str]:
        key = self._hash_context(context)
        if key in self.cache:
            return self.cache[key]["response"]
        return None

    def set(self, context: str, response: str):
        key = self._hash_context(context)
        self.cache[key] = {"response": response, "timestamp": now()}

效果

  • 重复查询成本降低90%
  • 响应延迟降低85%
  • API调用次数减少60%

四、最佳实践

4.1 上下文设计原则

原则说明
相关性只包含相关信息
简洁性避免冗余表述
结构化使用清晰格式
位置优化重要信息放首尾

4.2 上下文管理策略

上下文长度策略
短（<2000 tokens）精选最相关信息，重点信息置顶
中（2000-8000 tokens）结构化组织，分层次呈现
长（>8000 tokens）分块处理，智能压缩，动态加载

五、总结

上下文工程的核心要素：
  1. 理解模型行为：知道什么信息最有效
  2. 精心设计结构：组织信息的方式很重要
  3. 动态优化：根据场景调整策略
  4. 持续迭代：基于反馈改进上下文
