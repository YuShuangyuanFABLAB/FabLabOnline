Claude Code最佳实践

来源： https://www.anthropic.com/engineering/claude-code-best-practices
发布日期： 2025-04-18

一、环境设置

1.1 CLAUDE.md文件

CLAUDE.md是自动拉入上下文的特殊文件，Claude Code会在会话开始时自动读取它。

放置位置

位置适用场景
仓库根目录最常用，适用于大多数项目
运行目录的父目录Monorepo场景
~/.claude/CLAUDE.md全局配置，适用所有会话

二、常用工作流

2.1 探索-计划-编码-提交

Step 1: 探索 - 读取相关文件，不写代码
Step 2: 计划 - 制定方案，使用"think"
Step 3: 编码 - 实现解决方案
Step 4: 提交 - 创建commit，更新文档

2.2 具体化指令

糟糕良好
add tests for foo.pywrite a test case for foo.py covering the edge case where user is logged out
fix the bugthe login button doesn't work. investigate the click handler in Header.tsx

三、实战案例

案例 1

CLAUDE.md配置模板

场景

为一个Next.js电商项目创建CLAUDE.md，让Claude理解项目结构和规范。

解决方案

# E-Commerce Platform

## 常用命令
npm run dev          # 启动开发服务器
npm run test         # 运行测试
npm run db:migrate   # 数据库迁移

## 代码风格
- 使用函数组件和hooks
- 组件使用命名导出
- 使用Zod进行数据验证

## IMPORTANT规则
1. 所有数据库操作必须使用Prisma
2. 用户输入必须用Zod验证
3. 不要直接操作process.env

效果

Claude能快速理解项目，生成的代码符合规范，减少重复解释。

案例 2

自定义斜杠命令

场景

创建常用的代码审查、PR创建、测试编写快捷命令。

解决方案

在.claude/commands/创建Markdown文件：

# review.md
请对当前代码变更进行全面的代码审查：
- 代码质量（清晰度、命名、重复代码）
- 功能正确性（需求满足、边缘情况）
- 性能（循环、查询优化）
- 安全（输入验证、XSS/SQL注入）
- 测试（覆盖率、意义）

使用方式

/review - 执行代码审查
/pr - 创建PR
/test - 编写测试

案例 3

TDD工作流实践

场景

实现购物车功能，确保功能正确性和测试覆盖率。

工作流

  1. 编写测试：让Claude基于需求编写测试用例
  2. 运行测试：确认测试失败（红）
  3. 编写实现：让Claude编写代码通过测试
  4. 确认通过：所有测试变绿
  5. 提交：创建有意义的commit

效果

测试覆盖率高，代码质量有保证，回归测试方便。

案例 4

多Agent协作开发

场景

大型重构任务涉及多个模块，单一会话可能上下文溢出。

解决方案

# 创建多个worktree
git worktree add ../project-auth refactor/auth
git worktree add ../project-api refactor/api
git worktree add ../project-ui refactor/ui

# 不同终端启动Claude处理不同模块

效果

重构时间缩短60%，每个模块专注度高，冲突最小化。

案例 5

视觉迭代开发

场景

实现复杂UI组件，需要像素级还原设计稿。

工作流

  1. 提供设计稿：上传Figma截图
  2. Claude实现：创建初始版本
  3. 截图对比：识别与设计稿的差异
  4. 迭代修正：调整圆角、间距、颜色等
  5. 主题验证：测试深色/浅色主题
  6. 完成提交：确认后提交代码

效果

UI还原度高，减少返工，开发效率提升。

四、最佳实践总结

模糊的指令具体的指令（文件路径、行号）
缺少上下文提供相关背景信息
没有约束明确约束和边界
