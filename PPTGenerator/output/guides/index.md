## 二、核心原则

Anthropic AI方法论的核心可以概括为：

> **"寻找最简单的解决方案，只在需要时增加复杂度。"**

### 三大设计原则

| 原则 | 说明 |
|------|------|
| **简洁性** | 保持Agent设计简单，避免过度工程 |
| **透明性** | 明确显示Agent的规划步骤和决策过程 |
| **可测试性** | 精心设计Agent-Computer Interface (ACI) |

## 五、核心概念速查

### Agent类型

```
Workflows（工作流）
├── Prompt Chaining   # 串行步骤
├── Routing           # 分类路由
├── Parallelization   # 并行处理
├── Orchestrator-Workers  # 动态编排
└── Evaluator-Optimizer   # 迭代优化

Agents（自主代理）
└── 自主规划和执行，适合开放性问题
```

### 评估指标

| 指标 | 说明 |
|------|------|
| pass@k | k次尝试中至少成功一次的概率 |
| pass^k | 所有k次都成功的概率 |
| 评分器类型 | 代码/模型/人工 |

### 上下文策略

| 策略 | 适用场景 |
|------|----------|
| 压缩（Compaction） | 大量来回交互 |
| 结构化笔记 | 清晰里程碑的迭代开发 |
| 子代理架构 | 并行探索有价值的复杂研究 |
