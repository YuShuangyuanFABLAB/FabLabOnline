# 测试文件清理规则

## 规则

**默认保留最后3轮测试文件，仅在用户明确要求时删除所有临时文件。**

---

## 清理模式

### 模式1：常规清理（默认）
- 保留最近3轮测试生成的文件
- 删除更早的测试文件
- 适用于：功能开发中、问题排查中

### 模式2：完全清理（需用户确认）
- 删除所有临时测试文件
- 适用于：项目阶段结束、用户明确要求

---

## 需要管理的文件类型

### 1. 临时测试脚本
| 模式 | 说明 |
|------|------|
| `test_*.py` | 根目录下的临时测试脚本（非tests/目录） |
| `check_*.py` | 调试检查脚本 |
| `analyze_*.py` | 分析脚本 |
| `debug_*.py` | 调试脚本 |
| `verify_*.py` | 验证脚本 |

### 2. 输出文件
| 模式 | 说明 |
|------|------|
| `output/*.pptx` | 测试生成的PPT文件 |
| `output/*.xlsx` | 测试生成的Excel文件 |
| `output/*.pdf` | 测试生成的PDF文件 |
| `output/*.png` | 测试截图 |
| `output/*.json` | 测试数据文件 |

### 3. 保留的文件
| 文件 | 原因 |
|------|------|
| `tests/` 目录 | 正式的测试套件 |
| `output/.gitkeep` | 保持目录结构 |
| `test_runner.py` | 测试运行工具 |

---

## 清理命令模板

### 常规清理（保留最近3轮）

```bash
# 按时间排序，保留最近的文件，删除更早的
cd /path/to/project

# 保留最近3个PPTX文件
ls -t output/*.pptx 2>/dev/null | tail -n +4 | xargs rm -f

# 保留最近3个PNG文件
ls -t output/*.png 2>/dev/null | tail -n +4 | xargs rm -f

# 保留最近3个测试脚本
ls -t test_*.py check_*.py analyze_*.py 2>/dev/null | tail -n +4 | xargs rm -f
```

### 完全清理（需用户确认）

```bash
# 完全清理输出目录
rm -f output/*.pptx output/*.xlsx output/*.pdf output/*.png output/*.json
rm -rf output/screenshots/

# 完全清理临时脚本
rm -f analyze_*.py check_*.py debug_*.py verify_*.py test_*.py
```

---

## 判断"轮次"的方法

按文件修改时间判断：
1. 同一次会话中生成的文件视为同一轮
2. 按修改时间排序，最近的为最新轮次

---

## 检查清单

### 常规清理前
- [ ] 确认有超过3轮的测试文件
- [ ] 确认没有正在进行中的调试需要旧文件

### 完全清理前
- [ ] **用户已明确要求删除所有临时文件**
- [ ] 确认当前问题已彻底解决
- [ ] 确认不需要保留任何测试结果

### 清理后
- [ ] 确认保留了最近3轮文件（常规清理）
- [ ] 确认tests/目录完整保留

---

## 为什么要保留3轮

1. **回溯参考**：可能需要对比之前的测试结果
2. **调试需要**：问题可能未完全解决，需要参考历史
3. **安全边际**：避免误删正在使用的文件
