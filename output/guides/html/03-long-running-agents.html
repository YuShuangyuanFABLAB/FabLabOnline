长期运行Agent

来源： effective-harnesses-for-long-running-agents
发布日期： 2025-11-26

一、核心问题

Agent跨多个上下文窗口工作的挑战：

  • 无记忆：每个新会话开始时没有之前记忆
  • 上下文有限：单个会话能处理的内容有限
  • 复杂性：复杂项目无法在单个窗口内完成
关键洞察：让Agent一次只处理一个功能，通过结构化记录实现跨会话连续性。

二、两阶段解决方案

2.1 初始化Agent

设置初始环境，创建功能列表和进度追踪机制。

2.2 编码Agent

每个会话读取进度、选择功能、增量开发、记录进度。

三、功能列表设计

{
  "id": "F001",
  "category": "functional",
  "priority": "high",
  "description": "New chat button creates a fresh conversation",
  "steps": ["Navigate to main interface", "Click the 'New Chat' button"],
  "acceptance_criteria": ["Conversation list shows new entry"],
  "passes": false
}

四、实战案例

案例 1

电商平台开发

场景

使用长期运行Agent开发完整的电商平台，包含认证、产品目录、购物车等功能。

功能列表

F001: User registration and login (auth)
F002: Product catalog display (functional)
F003: Shopping cart functionality (functional)
F004: Checkout flow (functional)
F005: Order management (functional)

init.sh脚本

#!/bin/bash
npm install
cp .env.example .env
npm run db:migrate
npm run db:seed
npm run dev &
npm run test:e2e:smoke

效果

10个功能按计划完成，每个都有测试覆盖，代码可追溯。

案例 2

SaaS后台管理系统

场景

开发多租户SaaS后台，需要处理租户隔离和用户管理。

依赖管理

F001: Multi-tenant authentication (无依赖)
F002: Tenant isolation middleware (依赖F001)
F003: User management (依赖F001, F002)
F004: Admin dashboard (依赖F003)

效果

依赖关系清晰，不会遗漏前置条件，实现顺序合理。

案例 3

移动App后端API

场景

开发RESTful API服务，每个功能对应一组端点。

功能设计

API001: User endpoints (GET/POST/PUT/DELETE /users)
API002: Authentication endpoints (/auth/*)
API003: File upload endpoint (/upload, /files/:id)

API测试集成

init.sh包含newman运行API测试，确保端点正常工作。

案例 4

数据迁移工具

场景

开发数据库迁移工具，需要可恢复性。

检查点设计

{
  "migration_id": "mig_001",
  "progress": {
    "total_records": 1000000,
    "processed": 450000,
    "last_id": "user_450000"
  },
  "status": "in_progress"
}

效果

迁移可恢复，进度可见，失败可追踪。

案例 5

AI聊天机器人

场景

开发多功能AI聊天机器人，模块化设计。

功能模块

BOT001: Intent recognition (NLP)
BOT002: Entity extraction (NLP)
BOT003: Conversation state management (Dialogue)
BOT004: Slack integration
BOT005: Discord integration

跨会话一致性

每个会话记录上下文（使用的技术、已完成的模块），确保新会话能正确接续。

五、失败模式与解决方案

问题解决方案
过早宣布胜利设置严格的验收标准
环境遗留buginit.sh自动清理和重置
进度未记录强制每个会话更新进度文件
不知道如何运行init.sh脚本自动化

六、最佳实践

  1. 功能粒度：每个功能1-2个会话可完成
  2. 可测试性：有明确的验收标准
  3. 增量进度：每次只做一个功能
  4. 结构化记录：跨会话记忆
